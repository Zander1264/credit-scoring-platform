# Платформа автоматического кредитного скоринга (микросервисная архитектура)

Проект представляет собой **микросервисную систему**, разработанную в рамках курса **ШИФТ-Python**. Он реализует backend-платформу для обработки заявок на кредит, включая проверку на мошенничество, оценку кредитоспособности, маршрутизацию потоков и хранение данных.

Весь проект выполнен как **монорепозиторий** на **Python 3.12** с использованием фреймворка **FastAPI**. Каждый сервис независим, но взаимодействует с другими через HTTP и сообщения (Kafka), обеспечивая гибкость, масштабируемость и отказоустойчивость.

---

## 🧩 Микросервисы

### 1. `scoring-service` — Сервис Скоринга
Оценивает кредитоспособность пользователя на основе его данных:
- Анализирует финансовую историю, занятость, доход
- Принимает решение о возможности выдачи кредита
- Возвращает скоринговый балл и рекомендацию

### 2. `flow-selection-service` — Сервис маршрутизации
Определяет, к какой категории относится пользователь:
- Новый клиент
- Повторный заёмщик
- VIP-клиент (в будущем)
На основе этого выбирается дальнейший путь обработки заявки.

### 3. `data-service` — Сервис данных
Центральное звено для работы с данными:
- Взаимодействует с **PostgreSQL** через SQLAlchemy
- Хранит информацию о клиентах, заявках и кредитах
- Поддерживает миграции базы данных через **Alembic**
- Предоставляет REST API для других сервисов

### 4. `antifraud-service` — Сервис Антифрода
Обнаруживает признаки мошенничества:
- Проверка по "чёрным спискам"
- Анализ поведенческих паттернов
- Интеграция с внешними правилами (в будущем — ML)

## 🛠️ Технологии и инструменты

| Компонент        | Использование |
|------------------|-------------|
| **Python 3.12**  | Язык разработки |
| **FastAPI**      | Создание REST API, валидация данных, Swagger UI |
| **PostgreSQL**   | Хранение структурированных данных |
| **Redis**        | Кэширование, быстрый доступ к часто используемым данным |
| **Kafka**        | Асинхронное взаимодействие между сервисами (публикация/подписка) |
| **Docker**       | Контейнеризация каждого сервиса (`Dockerfile`) |
| **Kubernetes**   | Оркестрация (манифесты в `k8s/`) |
| **Helm**         | Управление развёртыванием (чарты в `helm/`) |
| **Alembic**      | Миграции базы данных |

> ⚠️ Примечание: Внешние инфраструктурные компоненты (Kafka, Redis, PostgreSQL) разворачивались на серверах курса. В репозитории — только код внутренних сервисов.

## 📁 Структура проекта

``` 
├── scoring-service # Оценка кредитного риска 
├── flow-selection-service # Определение типа клиента 
├── data-service # Работа с БД (PostgreSQL + Alembic) 
└── antifraud-service # Проверка на мошенничество
```

Каждый сервис включает:
- `src/app/api/` — HTTP-роуты (FastAPI)
- `src/app/monitoring/` — метрики, healthcheck, логирование
- `tests/` — юнит- и интеграционные тесты
- `Dockerfile` — для сборки образа
- `k8s/` и `helm/` — конфигурации для Kubernetes и Helm (кроме antifraud)

## 🚀 Особенности реализации

- ✅ **FastAPI** — использован во всех сервисах для создания быстрых, типизированных REST API с автоматической документацией (Swagger UI).
- ✅ **Асинхронная связь через Kafka** — сервисы обмениваются событиями (например: "заявка подана", "данные проверены"), что обеспечивает гибкость и отказоустойчивость.
- ✅ **Redis** — используется для кэширования часто запрашиваемых данных (например, справочников или результатов проверок).
- ✅ **PostgreSQL + Alembic** — в data-service реализована полноценная работа с реляционной БД: модели, сессии, миграции.
- ✅ **Тестирование** — каждый сервис покрыт автотестами (pytest), проверяющими корректность API и логики.
- ✅ **Контейнеризация** — каждый сервис имеет свой Dockerfile, готов к сборке и запуску в изолированной среде.
- ✅ **DevOps-интеграция** — конфигурации Kubernetes и Helm позволяют автоматизировать развёртывание и масштабирование.
- ✅ **CI/CD через GitLab CI** — реализована автоматическая проверка кода при каждом коммите:
  - Линтинг (black, isort, flake8)
  - Type-checking
  - Запуск тестов
  - Сборка Docker-образов
  - Подготовка к деплою
